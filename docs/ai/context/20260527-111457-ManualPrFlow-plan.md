# ManualPrFlow 计划

## 目标

把当前工作区改动从当前状态切到新分支，提交、推送到 GitHub，创建目标为 `main` 的 PR；完成客观自查后合并，并同步本地与远端 `main`。

## 范围

本次 PR 包含当前工作区内与以下工作相关的改动：

- `sensevoice-small` 后端隐藏切换与运行时测试。
- Ask Anything 选区链路诊断日志。
- 自动粘贴 app_compat 白名单扩展。
- 浏览器页面选区 UIA 前台窗口子树扫描。
- 对应测试、项目约束与上下文文档。

## 步骤

1. 从当前状态创建新分支。
2. 自查 diff 和文件范围，确认没有日志、构建产物或本地配置。
3. 运行相关测试与构建。
4. 显式 `git add` 需要提交的文件，忽略运行日志和本地缓存。
5. 提交并推送新分支。
6. 创建目标分支为 `main` 的 PR。
7. 做一次客观 review，确认风险和验证结果。
8. 合并 PR。
9. 切回本地 `main`，拉取远端最新代码，清理临时分支。
