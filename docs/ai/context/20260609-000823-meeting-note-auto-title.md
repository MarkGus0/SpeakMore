# 会议笔记内容自动命名

## 背景

会议笔记列表当前会把录制入口名直接保存为笔记标题，例如“新笔记”“实时翻译”，导入文件则保留原始文件名。用户希望列表卡片里的标题根据笔记内容生成一个更合适的名字，而不是继续显示模块默认名。

## 设计取舍

- 不新增后端接口，不为标题单独调用大模型，避免录制结束后额外等待和失败点。
- 在 renderer 增加纯函数，根据最终结构化纪要、摘要、转写和译文做本地提炼。
- 只替换默认标题或弱标题，例如“新笔记”“实时翻译”“会议标题”“Meeting title”和导入文件名；用户手动编辑过的标题不覆盖。
- 标题优先级为：结构化主题标题、结构化摘要、决策/待办/行程、最终摘要、逐字稿、译文。
- 标题长度控制为桌面列表友好的短标题：中文优先 8-24 字左右，英文约 4-8 个词；去掉 emoji、Markdown 标题、序号、常见前缀和尾部标点。
- 内容太少时仍避免回退到模块名，使用带日期时间的“会议记录”类标题。

## 验证方式

- 为标题 helper 增加单元测试，覆盖默认标题替换、手动标题保留、结构化主题优先、文件名弱标题替换、内容不足 fallback。
- 运行 renderer 测试和构建，确保会议笔记页面仍能通过 TypeScript 和现有结构测试。
- 运行主进程语法检查和 `git diff --check`。
