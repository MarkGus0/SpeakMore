# 本地翻译模型设置区块精简

## 背景

用户反馈本地翻译模型卡片中的“运行方式”和“运行说明”两行显得臃肿。截图中红框标出了：

- `运行方式 / 稳定模式`
- `运行说明 / 极速运行时未就绪，已自动使用稳定模型。`

当前卡片还会在 ready 状态下展示后端 detail 文案，进一步增加视觉负担。

## 设计

- 删除设置页本地翻译模型卡片里的运行方式行。
- 删除设置页本地翻译模型卡片里的运行说明/fallback reason 行。
- ready 状态不再展示额外 detail 文案，只保留绿色“已加载”状态。
- 失败、下载中断、运行时缺失、模型加载失败等异常状态仍显示 i18n 说明，避免用户无法处理错误。
- 不改变后端状态字段、模型下载/加载/自动接管 existing llama-server 的真实功能。

## 验证

- 更新 renderer 结构测试，确保本地翻译模型区块不再引用 runtime mode/fallback reason UI 文案。
- 运行 renderer 测试、renderer build、主进程语法检查和 diff check。
