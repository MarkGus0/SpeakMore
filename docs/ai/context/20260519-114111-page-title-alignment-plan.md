# 页面标题对齐设计与计划

## 背景

设置页标题“设置”当前位于内容面板左上角，根容器为 `p: 3`，标题为 `fontSize: 24`、`fontWeight: 500`。其他页面部分根容器使用 `mx: 'auto'` 居中，导致切换页面时“首页”“历史记录”等一级标题的左侧坐标与设置页不一致。

## 目标

- 以设置页标题的位置和字号作为标准，统一首页、历史记录、词典、模型、诊断五个页面的一级标题。
- 保持各页面已有内容、交互和文案不变，避免扩大到整页重构。

## 方案

推荐方案：在 `src/uiTokens.ts` 增加共享 `pageSx` 和 `pageTitleSx`。

- `pageSx` 固定页面根容器左上基准：`p: 3`，不使用 `mx: 'auto'`。
- 各页面可在 `pageSx` 基础上保留自己的 `maxWidth`、布局方向和高度。
- `pageTitleSx` 固定一级标题样式：`fontSize: 24`、`fontWeight: 500`。

不采用的方案：

- 逐页手写相同样式：改动少但后续容易再次漂移。
- 调整 AppShell 统一内边距：会影响所有页面滚动和内容布局，范围过大。

## 实施步骤

1. 增加结构测试，锁定共享标题 token 和五个页面不再使用 `mx: 'auto'` 偏移标题。
2. 在 `uiTokens.ts` 增加 `pageSx`、`pageTitleSx`。
3. 修改首页、历史记录、词典、模型、诊断五个页面，引入共享 token。
4. 同步更新 `AGENTS.md` 的长期前端约束。
5. 运行 `cd electron-app/renderer; npm test` 和 `npm run build`。

## 验证

- 测试应覆盖标题结构约束。
- 前端构建必须通过，确保 TS 与 Vite 产物正常。
